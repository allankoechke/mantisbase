Healthcheck Endpoint

The healthcheck endpoint provides a simple way to verify that the MantisBase server is running and responsive. It is useful for monitoring tools, load balancers, and deployment scripts.

Endpoint

GET /api/v1/health

Response

A successful request returns a JSON response with the following structure:

{
"status": "ok",
"uptime": 12345
}
  • status: Always set to "ok" if the server is healthy
  • uptime: Number of seconds the server has been running since startup

The HTTP status code for a healthy response is 200 OK.

Example

Request:

curl http://localhost:7070/api/v1/health

Response:

{
"status": "ok",
"uptime": 12345
}

Error Handling

If the server encounters an error while processing the healthcheck, it may return a 500 Internal Server Error with an appropriate error message.

Usage

  • Verify server availability and responsiveness
  • Integrate with monitoring systems (Prometheus, Nagios, etc.)
  • Use in orchestration systems (Kubernetes liveness/readiness probes, Docker healthchecks)
  • Load balancer health checks

Example Docker healthcheck:

HEALTHCHECK --interval=30s --timeout=3s \
CMD curl -f http://localhost:7070/api/v1/health || exit 1

Example Kubernetes liveness probe:

livenessProbe:
httpGet:
path: /api/v1/health
port: 7070
initialDelaySeconds: 30
periodSeconds: 10

Summary

The healthcheck endpoint provides a simple, lightweight way to monitor MantisBase server status and uptime, making it easy to integrate with various monitoring and orchestration systems.
